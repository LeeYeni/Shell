주제: “프로젝트 폴더 자동 생성 & 정리 스크립트”

🎯 목표
하나의 쉘 스크립트 파일을 실행하면, 기본 프로젝트 디렉토리 구조와 템플릿 파일을 자동으로 만드는 것.

✅ 스펙/요구사항

다음과 같은 폴더 구조 생성 (프로젝트이름이 예를 들어 demo_project라고 하면):

demo_project/
├── data/
│ ├── raw/
│ └── processed/
├── notebooks/
├── src/
├── reports/
├── README.md
└── .gitignore

각 디렉토리/파일에 대한 처리
이미 동일 이름의 폴더가 존재하면,
“이미 존재합니다”라고 메시지만 출력하고 덮어쓰지는 않기.

README.md 파일이 없으면 새로 만들고, 다음 내용(형식만, 실제 문구는 자유)을 자동으로 적어 넣기:
프로젝트 이름
오늘 날짜 (쉘에서 날짜 명령 사용)
간단한 한 줄 설명 (고정된 텍스트로 괜찮음)
실행이 끝나면,
생성된(또는 이미 있던) 디렉토리와 파일 목록을 한 번에 출력해주기.

🧩 단계별 가이드
#!/bin/bash로 시작하는 스크립트 파일 만들기.
인자 개수 체크:
$#를 사용해서 인자가 1개가 아니면 사용법 출력 후 종료.
첫 번째 인자를 프로젝트 이름 변수에 저장 (PROJECT_NAME=$1).
mkdir -p를 사용해서 필요한 디렉토리를 모두 생성.
README.md가 존재하는지 [ -f ... ]로 확인 후:
없으면 echo와 리다이렉션(>, >>)을 사용해서 내용 쓰기.
마지막에 tree 또는 find 명령으로 전체 구조를 출력 (만약 tree가 없다면 find 사용).
-f 옵션을 주(예: ./init_project.sh -f 프로젝트이름) 기존 폴더가 있어도 “경고 후 강제 진행”하도록 옵션 처리해보기.

logs/ 디렉토리와 간단한 .gitignore 파일까지 자동 생성해보기.
